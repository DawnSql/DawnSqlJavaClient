基础概念

SuperSql 的基础表

my_users_group (用户组) 对应于企业的业务部门
my_user(用户) 对应于企业业务部门的一个小分组或者是一个模块
可以简单的理解为，用户组对应于部门，而任务组对应于部门下面的分组或者模块
SuperSql 有一个独创的概念称为场景
它表示：一个拥有输入、输出的 sql 语句的集合，而这个集合相当于一个 sql 语句的内置方法可以被其它 sql 所使用。同时可以对这个场景设置相应的权限

开发过程

  1. 首次开发先添加用户组、在根据用户组添加用户。
    用户组的作用是：为用户组添加表和场景的增删改查的权限。
    用户必须属于一个用户组。
  1. 创建表，创建视图， 创建场景分配到任务组

添加用户组

xxxxxxxxxx
INSERT INTO my_users_group (group_name) VALUES ('用户组的名字');
-- 例如：
-- INSERT INTO my_users_group (group_name) VALUES ('流程优化一组');
-- INSERT INTO my_users_group (group_name) VALUES ('流程优化二组');
-- INSERT INTO my_users_group (group_name) VALUES ('管理员');

添加用户

xxxxxxxxxx
-- 添加用户分两步
-- 1、查询要添加用户的用户组 id, 既 group_id
SELECT m.id AS group_id FROM my_users_group AS m WHERE m.group_name = '流程优化一组';
-- 2、添加用户 
-- user_name, pass_word, group_id 添加到 my_user
INSERT INTO my_user (id, user_name, pass_word, group_id) VALUES (auto_id('my_users_group'), '用户名字', '密码', 1);

添加任务组

xxxxxxxxxx
INSERT INTO my_task_group (task_group_name) VALUES ('任务组的名字');

添加用户到任务组

xxxxxxxxxx
-- 1、查询任务组的 task_group_id (例如：task_group_id = 10)
SELECT m.id AS task_group_id FROM my_task_group AS m WHERE m.task_group_name = '任务组名';
​
-- 2、查询用户的 user_id (例如：user_id = 100)
SELECT m.id AS user_id FROM my_user AS m WHERE m.user_name = '任务组名';
​
-- 3、将用户添加到任务组
INSERT INTO my_group (task_group_id, user_id) VALUES (10, 100);

创建场景

创建一个场景、并将场景分配到任务组。然后在工作台，完成场景的创作和测试。

xxxxxxxxxx
INSERT INTO my_scenes (scenes_name, version, status) VALUES ('场景名称', '版本号', '开发');
​
-- 将场景分配给任务组
-- 1、查询任务组的 task_group_id (例如：task_group_id = 10)
SELECT m.id AS task_group_id FROM my_task_group AS m WHERE m.task_group_name = '任务组名';
​
-- 2、查询 my_scenes 的 scenes_id (例如：scenes_id = 102)
SELECT m.id AS scenes_id FROM my_scenes AS m WHERE m.scenes_name = '场景名称' and m.version = '版本号' and m.status = '开发';
​
-- 3、添加到任务列表
INSERT INTO my_task (task_group_id, scenes_id) VALUES (10, 102);

创作场景

场景分为两大类：
1、实时场景，表示执行 sql 集合返回的数据集小。默认为小于等于 1000 条数据集
2、批量场景，表示执行 sql 集合返回的数据集大。默认为大于 1000 条数据集。对于批量场景需要先创建一个表来保存场景计算是数据集，使用的时候，直接读取这个数据集。
